# รายงานเปรียบเทียบ SEO: เว็บไซต์เดิม vs เว็บไซต์ใหม่

**โครงการ:** Ryoko Tackle (ร้านอุปกรณ์ตกปลา)
**วันที่จัดทำ:** 4 มิถุนายน 2026
**เครื่องมือตรวจ:** Seobility (เครื่องมือเดียวกันทั้งสองเว็บ — เทียบกันได้โดยตรง)
**เว็บเดิม:** `https://www.ryokotackle.com/index.php` (ReadyPlanet)
**เว็บใหม่:** `https://ryokotackle.jayjaygt1098.workers.dev/` (Next.js 16 + Cloudflare Workers)

---

## 1. บทสรุปสำหรับผู้บริหาร

คะแนน On-page SEO รวมเพิ่มขึ้นจาก **48% → 56%** แต่ตัวเลขรวมนี้ **ยังไม่สะท้อนของจริง** เพราะเว็บใหม่ถูกตรวจบน **โดเมนชั่วคราว** (`workers.dev`) ที่ยังไม่ได้เชื่อมโดเมนจริง `ryokotackle.com` ทำให้ถูกหักคะแนนจากหลายข้อที่ **จะหายไปเองเมื่อขึ้นโดเมนจริง**

ภาพรวมที่แท้จริง:
- ✅ **ด้านโครงสร้าง (Structure) พุ่งจาก 58% → 100%** และ **ลิงก์ (Links) จาก 25% → 87%** — ปัญหาระดับ Error ที่ร้ายแรงของเว็บเดิม (ไม่มี H1, ไม่มี heading, รูปไม่มี alt, แท็ก bold ขยะ 326 อัน, URL dynamic) ถูกแก้ **หมดทุกข้อ**
- ⚠️ **ด้าน Meta data กลับลดลง 61% → 43%** เพราะ **title และ meta description ยาวเกินขีดจำกัด (พิกเซล)** — เป็นจุดที่ต้องแก้ในโค้ด (แก้ได้เร็ว)
- ⚠️ **เวลาตอบสนองช้า 1.89 วินาที** — เกิดจาก cold start บนโดเมนชั่วคราว + เซิร์ฟเวอร์ทดสอบอยู่เยอรมนี จะดีขึ้นมากเมื่อใช้โดเมนจริง + Cloudflare cache
- ⚪ **Backlink ยังคง 3% เท่าเดิม** — เป็นงานการตลาด ไม่ใช่งานโค้ด

## 2. เปรียบเทียบคะแนนรายหมวด (ตัวเลขจริงทั้งสองฝั่ง)

| หมวด | เว็บเดิม | เว็บใหม่ | การเปลี่ยนแปลง |
|---|---|---|---|
| **คะแนนรวม On-page** | 48% | **56%** | 🟢 +8 |
| Page structure (โครงสร้าง) | 58% | **100%** | 🟢 +42 |
| Links (ลิงก์) | 25% | **87%** | 🟢 +62 |
| Server (เซิร์ฟเวอร์) | 70% | **87%** | 🟢 +17 |
| Page quality (คุณภาพหน้า) | 35% | **54%** | 🟢 +19 |
| Meta data (ข้อมูล meta) | 61% | **43%** | 🔴 −18 |
| External factors (backlink) | 3% | **3%** | ⚪ คงที่ |

> สรุป: 5 ใน 6 หมวดดีขึ้น โดยเฉพาะหมวดโครงสร้างและลิงก์ที่เคยแย่ที่สุดกลับดีขึ้นมากที่สุด มีเพียงหมวด Meta data ที่ลดลงจากปัญหาความยาว title/description ซึ่งแก้ได้ทันที

## 4. ปัญหาที่เหลือบนเว็บใหม่ (แยกตามประเภท)

### 4.1 🔧 แก้ได้ในโค้ด (ควรทำทันที)

| ปัญหา (Seobility) | รายละเอียด | แนวทางแก้ |
|---|---|---|
| **Title ยาวเกิน** (752px > 580px) | ตัวอักษรไทยกว้างกว่าอังกฤษในเชิงพิกเซล | ย่อ title หน้าแรกให้สั้นลง เช่น "Ryoko Tackle \| อุปกรณ์ตกปลา คันเบ็ด รอก เหยื่อ" |
| **Meta description ยาวเกิน** (1482px > 1000px) | คำอธิบายไทยยาวเกินงบพิกเซล | ตัดให้เหลือ ~120 ตัวอักษรไทย ตัดรายชื่อแบรนด์บางส่วนออก |
| **X-Powered-By header** (Warning) | Next.js ส่ง header เปิดเผยเทคโนโลยี | ตั้ง `poweredByHeader: false` ใน `next.config.ts` |
| **H1 ไม่ตรงเนื้อหา** (Warning) | คำใน H1 (ไทย) ไม่ปรากฏในเนื้อหาหน้าแรก | เพิ่มเนื้อหาไทยในหน้าแรก / ปรับ H1 ให้ตรงกับข้อความที่แสดงจริง |
| **เนื้อหาน้อย** (208 คำ, ควร ≥250) | หน้าแรกเน้นรูปภาพ มีข้อความน้อย | เพิ่มย่อหน้าแนะนำร้าน/หมวดสินค้าเป็นภาษาไทย |
| **ไม่มี Apple touch icon** (Warning) | — | เพิ่ม `apple-icon.png` ใน `app/` |

### 4.2 🌐 จะหายเองเมื่อใช้โดเมนจริง `ryokotackle.com`

| ปัญหา (Seobility) | สาเหตุ |
|---|---|
| **Canonical ชี้คนละโดเมน** (Error) | canonical ตั้งเป็น `www.ryokotackle.com` (ถูกต้องสำหรับ production) แต่กำลังทดสอบบน `workers.dev` — จะตรงกันเมื่อ deploy โดเมนจริง |
| **โฮสต์บน subdomain** (Warning) | `workers.dev` เป็น subdomain — โดเมนจริงเป็น top-level domain |
| **HTTPS redirect ไม่ถูกตั้งค่า** (Error) | จัดการอัตโนมัติโดย Cloudflare เมื่อผูกโดเมนจริง |
| **เวลาตอบสนอง 1.89 วิ** (Error) | cold start ของ Worker + ทดสอบจากเยอรมนี + ยังไม่มี cache — โดเมนจริง + Cloudflare CDN จะเร็วขึ้นมาก |

### 4.3 🌍 ปัจจัยภายนอก/เนื้อหา (นอกเหนือขอบเขตโค้ด)

| ปัญหา | หมายเหตุ |
|---|---|
| **Backlink 3%** (1 ลิงก์ 1 โดเมน) | ต้องสร้างลิงก์จากพันธมิตร, marketplace, โซเชียล, สื่อตกปลา |

## 5. แผนการดำเนินงานถัดไป (เรียงตามความสำคัญ)

1. **(ก่อน/หลัง deploy ทันที)** ย่อ title และ meta description ให้อยู่ในงบพิกเซล + ปิด X-Powered-By → ดึงคะแนน Meta data กลับขึ้น
2. **ผูกโดเมนจริง `ryokotackle.com`** เข้ากับ Cloudflare Worker → แก้ canonical, subdomain, HTTPS redirect, และเวลาตอบสนองพร้อมกัน
3. **เพิ่มเนื้อหาภาษาไทยหน้าแรก** (≥250 คำ) ให้ H1 และ title ตรงกับเนื้อหา → ดึงคะแนน Page quality
4. **ทำ 301 Redirect** จาก URL เดิม 408 รายการ → slug ใหม่ เพื่อรักษาอันดับ/พลังลิงก์
5. **สร้าง Backlink** ผ่านการตลาด (งานต่อเนื่อง)

## 6. วิธีการตรวจสอบ (Methodology)

ทั้งสองเว็บถูกตรวจด้วย **Seobility** เครื่องมือเดียวกัน จึงเปรียบเทียบกันได้โดยตรง

| | เว็บเดิม | เว็บใหม่ |
|---|---|---|
| URL | ryokotackle.com/index.php | ryokotackle.jayjaygt1098.workers.dev |
| คะแนนรวม | 48% | 56% |
| จำนวนคำ | 975 | 208 |
| ขนาดไฟล์ HTML | 169 kB | 82 kB |
| เวลาตอบสนอง | 0.78 วิ | 1.89 วิ (cold start) |
| Issues วิกฤต | 6 | 4 |

> **ข้อควรทราบ:** คะแนนเว็บใหม่วัดบนโดเมนชั่วคราว เมื่อแก้รายการในข้อ 5.1–5.2 และตรวจซ้ำบนโดเมนจริงแล้ว คาดว่าคะแนนรวมจะสูงกว่า 56% อย่างมีนัยสำคัญ (โดยเฉพาะหมวด Meta data และ Server)
